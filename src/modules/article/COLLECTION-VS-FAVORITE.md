# 收藏 vs 合集（收藏夹）

## 概念区分

### 收藏（Favorite）
- **用途**：用户收藏喜欢的文章
- **特点**：简单的收藏关系，类似点赞
- **创建者**：任何用户
- **可见性**：私有
- **数据表**：`article_favorite`
- **接口**：`/article/:id/favorite`

### 合集/收藏夹（Collection）
- **用途**：作者创建文章集合，组织系列文章
- **特点**：可排序、可分类、可公开展示
- **创建者**：通常是作者
- **可见性**：可公开可私有
- **数据表**：`favorite` + `favorite_item`
- **接口**：`/favorite`

## 使用场景

### 收藏
```
用户看到喜欢的文章 → 点击收藏按钮 → 文章被收藏
```
- 快速收藏，无需选择收藏夹
- 用于个人阅读记录
- 类似书签功能

### 合集
```
作者创建系列文章 → 创建合集 → 添加文章到合集 → 公开展示
```
- 需要组织和管理
- 用于展示相关文章
- 类似专辑/播放列表

## 数据结构对比

### 收藏
```
article_favorite
├── id
├── userId (收藏者)
├── articleId
└── createdAt
```

### 合集
```
favorite (收藏夹/合集)
├── id
├── name
├── description
├── userId (创建者)
├── isPublic
└── itemCount

favorite_item (合集中的文章)
├── id
├── favoriteId
├── articleId
├── userId
├── sort (排序)
└── note (备注)
```

## API 对比

### 收藏 API
```
POST   /article/:id/favorite        # 收藏文章
DELETE /article/:id/favorite        # 取消收藏
GET    /article/:id/favorite/status # 检查收藏状态
GET    /article/favorited/list      # 获取收藏列表
```

### 合集 API
```
POST   /favorite                    # 创建合集
GET    /favorite                    # 获取合集列表
GET    /favorite/:id                # 获取合集详情
PATCH  /favorite/:id                # 更新合集
DELETE /favorite/:id                # 删除合集
POST   /favorite/add                # 添加文章到合集
DELETE /favorite/:id/article/:aid   # 从合集移除文章
GET    /favorite/:id/items          # 获取合集中的文章
```

```
┌─────────────────────────────────┐
│ 作者信息                         │
```

```
┌─────────────────────────────────┐

│ ├─ 文章2                        │

├─────────────────────────────────┤
│ 我的合集                         │  ← 用户创建的合集
```

## 实现建议

1. **收藏按钮**：放在文章详情页的操作栏，与点赞、评论并列
2. **合集入口**：放在作者主页或文章详情页的侧边栏
3. **图标区分**：
   - 收藏：⭐ 星标图标
   - 合集：📚 书籍/文件夹图标
4. **文案区分**：
   - 收藏：收藏/取消收藏
   - 合集：添加到合集/创建合集
